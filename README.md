## 🧑‍💻 About Me

I design and build **data infrastructure** and **AI-powered automation systems** — from raw data ingestion to production-grade pipelines with intelligent error handling. My work spans the full data engineering stack, with a growing focus on applied AI: agent orchestration, LLM integration, and workflow automation.

I approach every system with three priorities: **reliability**, **scalability**, and **maintainability**. Whether it's designing ETL architecture, building AI agents with LangGraph, or wiring distributed Spark jobs, the goal is always clean, production-ready output.

Currently building: an **Automated ETL Pipeline with AI-Powered Error Detection and Monitoring Dashboard** — a production system that uses ML anomaly detection to catch pipeline failures before they reach downstream systems.

### Automated ETL Pipeline with AI-Powered Error Detection

A production-grade data pipeline system built with intelligent monitoring at its core. Instead of passive logging, the system uses ML-based anomaly detection to identify failures, data drift, and schema issues in real time — before they affect downstream consumers.

**What it does:**
- Ingests data from multiple heterogeneous sources automatically
- Detects anomalies and pipeline errors using trained ML models
- Surfaces issues on a real-time monitoring dashboard with severity levels
- Triggers automated alerts and configurable self-healing workflows
- Designed for reliability, observability, and production deployment
